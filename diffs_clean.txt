-
-PURPOSE
--------
-Provides admin visibility into which user modules are assigned to which base stations, based on the operator who registered them. This creates a hierarchical organizational structure for better monitoring and coordination.
-
-
-CORE CONCEPT
-------------
-The module uses an existing relationship chain without requiring new database fields:
-
-User Module (DEV_001)
-  ↓ registered_by_emp_id
-Operator (EMP001)
-  ↓ assigned_station
-Base Station (Station-A)
-
-This means:
-- When operator EMP001 registers device DEV_001
-- And EMP001 is assigned to Station-A
-- Then DEV_001 is implicitly assigned to Station-A
-
-

-Step 1: Operator Registers Device

-1. Project Overview
-Project Name: Forest Emergency Communication System (FECS)
-Purpose: FECS is an offline emergency communication system for forest and remote terrain environments. It allows a user module carried by trekkers or forest personnel to send alert messages through LoRa repeater nodes to a base station. The base station is connected to a laptop/server that runs the backend and dashboard application.
-The application layer is responsible for receiving alerts, validating them, removing duplicates, storing valid records, showing them on a live dashboard, and triggering acknowledgments back to the user module through the base station.
-________________________________________
-2. High-Level Architecture
-2.1 Physical Modules
-1. User Module
-•	Contains push button, GPS, LoRa, ESP32, LED, buzzer
-•	Sends alert messages based on button interaction
-•	Receives acknowledgment from the base station through relay nodes
-2. Relay Module
-•	Works as a repeater/forwarding node
-•	Receives LoRa packets and retransmits them
-•	Extends network coverage between user module and base station
-•	Does not perform backend logic
-3. Base Station Module
-•	Connected to laptop/server
-•	Receives messages from user modules through relay nodes
-•	Forwards structured data to backend
-•	Receives ACK commands from backend and transmits ACK back through relay nodes to user module
-2.2 Logical Data Flow
-User Module -> Relay Module(s) -> Base Station Module -> Backend -> Database -> Dashboard
-Backend -> Base Station Module -> Relay Module(s) -> User Module
-________________________________________
-3. Application Scope
-The software application includes: - Backend service using FastAPI - Frontend dashboard using React.js - Live map using Leaflet - PostgreSQL database for alert storage - MQTT-based communication between base station and backend
-________________________________________

-Backend
-•	FastAPI
-•	Pydantic validation
-•	MQTT client
-•	PostgreSQL driver / ORM
-Frontend
-•	React.js

-•	Real-time updates from backend
-Database
-•	PostgreSQL
-Communication

-5.1 Alert Reception

-5.2 Message Types
-Supported message types: - normal - high - emergency - cancel
-5.3 Signal Types
-Supported signal types: - manual - auto

-•	do not store duplicate entries

-## Alert Message Format
-
-**Pipe-Delimited Format** (52 bytes):
-```
-test001|DEV_001|10.9347|78.1212|E|M|1705315800|1|A3F2
-```
-
-**Field Order**:
-1. packet_id (unique identifier)
-2. device_id (hardware device ID)
-3. latitude (GPS coordinate)
-4. longitude (GPS coordinate)
-5. message_type: `E` (Emergency), `W` (Warning), `I` (Info)
-6. signal_type: `M` (Manual), `A` (Automatic)
-7. event_time (Unix timestamp)
-8. source: `1` (Hardware), `0` (Software)
-9. checksum (optional CRC16)
-
-**Why Pipe-Delimited?**
-- 70% smaller than JSON (~52 bytes vs ~180 bytes)
-- Lower packet loss in weak signal areas
-- Faster ESP32 processing (no JSON parsing)
-- Better for LoRa/RF communication

+# User Management Toggle Fix
+
+## Problem
+The toggle button in User Management was not working correctly because of a data type mismatch:
+- **Database**: Stores `is_active` as INTEGER (0 or 1)
+- **Frontend**: Was treating it as BOOLEAN (true/false)
+- **Backend Schema**: Expected BOOLEAN but database uses INTEGER
+
+## Solution Applied
+
+### 1. Frontend Fix (`UserManagement.jsx`)
+
+**Before:**
+```javascript
+const toggleUserStatus = async (userId, currentStatus) => {
+  await axios.patch(
+    `http://localhost:8000/api/users/${userId}`,
+    { is_active: !currentStatus },  // Boolean negation doesn't work with 0/1
+    { headers: { Authorization: `Bearer ${token}` } }
+  );
+};
+```
+
+**After:**
+```javascript
+const toggleUserStatus = async (userId, currentStatus) => {
+  // Convert to integer properly - currentStatus is 1 or 0 from database
+  const newStatus = currentStatus === 1 ? 0 : 1;
+  
+  await axios.patch(
+    `http://localhost:8000/api/users/${userId}`,
+    { is_active: newStatus },  // Send integer value
+    { headers: { Authorization: `Bearer ${token}` } }
+  );
+  
+  alert(`User ${newStatus === 1 ? 'activated' : 'deactivated'} successfully!`);
+};
+```
+
+**Display Fix:**

If the user fails to acknowledge this challenge within 60 seconds, the software automatically escalates their status to an 'Emergency' on the dashboard.
-
-2. WHY IS THIS FEATURE NECESSARY?
----------------------------------
-In remote or hazardous forest environments, traditional SOS systems rely entirely on the victim being conscious and physically capable of pressing an emergency button. 
-
-*   The Blind Spot: If a ranger falls, is attacked by wildlife, or suffers a sudden medical emergency (e.g., snakebite, heart attack) and loses consciousness, they cannot press the SOS button. 
-*   The Solution: This feature acts as an automated "Dead Man's Switch." It shifts the system from passive listening to active monitoring. It ensures that an incapacitated ranger is never left behind simply because they couldn't call for help themselves.
-
-3. THE ACK MESSAGE FORMAT (SERVER -> BASE -> USER MODULE)
----------------------------------------------------------
-Because the ESP32 parser requires strict adherence to bandwidth and memory constraints, the outbound Acknowledgment (ACK) message from the server strips out heavy JSON and unused timestamp fields, favoring an ultra-lightweight pipe-delimited format.
-
-Final Format Design:
-[packet_id] | [device_id] | [ack_type]
-
-Field Breakdown:
-1. packet_id: A unique identifier generated by the server (e.g., "ACK_17203..."). Tells the hardware not to process this ping twice if relayed multiple times.
-2. device_id: The specific hardware identifier (e.g., "DEV_001"). Ensures that only the targeted ranger's buzzer activates, not the entire forest network.
-3. ack_type: A critical single-character flag parsed by the ESP32.
-   - 'L' = LED Blink (Quiet confirmation that a normal message was received by the server).
-   - 'B' = Buzzer Alarm (Loud challenge fired by the server demanding a manual button press response).
-

-ACK_17109283|DEV_001|B

+import React, { useState } from 'react';
+import { MdExpandMore, MdExpandLess, MdHistory, MdNotificationsActive } from 'react-icons/md';
+import { FaCheckCircle, FaCheck } from 'react-icons/fa';
+import { updateAlertStatus } from '../services/api';
+import './ExpandableAlertList.css';
+
+function ExpandableAlertList({ alerts, token, onUpdate }) {
+  const [expandedDevices, setExpandedDevices] = useState({});
+  const [deviceHistory, setDeviceHistory] = useState({});
+  const [loadingHistory, setLoadingHistory] = useState({});
+  const [notes, setNotes] = useState({});
+
+  const toggleExpand = async (deviceId) => {
+    const isExpanded = expandedDevices[deviceId];
+    
+    setExpandedDevices({
+      ...expandedDevices,
+      [deviceId]: !isExpanded
+    });
+
+    // Fetch history if expanding and not already loaded
+    if (!isExpanded && !deviceHistory[deviceId]) {
+      setLoadingHistory({ ...loadingHistory, [deviceId]: true });
+      try {
+        const response = await fetch(`http://localhost:8000/api/alerts/device/${deviceId}`, {
+          headers: {
+            'Authorization': `Bearer ${token}`
+          }
+        });
+        const history = await response.json();
+        setDeviceHistory({ ...deviceHistory, [deviceId]: history });
+      } catch (error) {
+        console.error('Failed to fetch device history:', error);
+      } finally {
+        setLoadingHistory({ ...loadingHistory, [deviceId]: false });
+      }
+    }
+  };
+
+  const handleStatusChange = async (alertId, newStatus, alertNotes) => {
+    try {
+      await updateAlertStatus(alertId, newStatus, alertNotes, token);
+      onUpdate();
+    } catch (error) {
+      console.error('Failed to update status:', error);
+    }
+  };
+
+  const getAlertClass = (messageType) => {
+    switch (messageType) {
+      case 'emergency': return 'alert-item emergency';
+      case 'high': return 'alert-item high';
+      case 'normal': return 'alert-item normal';
+      default: return 'alert-item';
+    }
+  };
+
+  const getAlertIcon = (messageType) => {
+    switch (messageType) {
+      case 'emergency': return '🔴';
+      case 'high': return '🟠';
+      case 'normal': return '🟢';
+      default: return '🔵';
+    }
+  };
+
+  const formatTimeAgo = (timestamp) => {
+    const now = new Date();
+    const alertTime = new Date(timestamp);
+    const diffMs = now - alertTime;
+    const diffMins = Math.floor(diffMs / 60000);
+    const diffHours = Math.floor(diffMins / 60);
+    const diffDays = Math.floor(diffHours / 24);
+
+    if (diffMins < 1) return 'Just now';
+    if (diffMins < 60) return `${diffMins} min ago`;
+    if (diffHours < 24) return `${diffHours} hour${diffHours > 1 ? 's' : ''} ago`;
+    return `${diffDays} day${diffDays > 1 ? 's' : ''} ago`;
+  };
+
+  return (
+    <div className="expandable-alert-list">
+      <h2>
+        <MdNotificationsActive className="alert-list-icon" />
+        Active Personnel ({alerts.length})
+      </h2>
+      {alerts.length === 0 ? (
+        <p className="no-alerts">No active personnel to display</p>
+      ) : (
+        <div className="alert-cards">
+          {alerts.map((alert) => {
+            const isExpanded = expandedDevices[alert.device_id];
+            const history = deviceHistory[alert.device_id] || [];
+            const isLoading = loadingHistory[alert.device_id];
+            const currentNotes = notes[alert.id] || alert.notes || '';
+
+            return (
+              <div key={alert.device_id} className="alert-card-wrapper">
+                {/* Main Alert Card */}
+                <div className={getAlertClass(alert.message_type)}>
+                  <div className="alert-header" onClick={() => toggleExpand(alert.device_id)}>
+                    <div className="alert-user-info">
+                      <span className="alert-icon">{getAlertIcon(alert.message_type)}</span>
+                      <div className="user-details">
+                        <span className="user-name">
+                          {alert.user_name || alert.device_id}
+                          {!alert.user_name && (
+                            <span className="unregistered-badge">⚠️ Unregistered</span>
+                          )}
+                        </span>
+                        <span className="device-id-small">{alert.device_id}</span>
+                      </div>
+                    </div>
+                    <div className="alert-meta">
+                      <span className="alert-type">{alert.message_type}</span>
+                      <span className="alert-time">{formatTimeAgo(alert.received_at)}</span>
+                      <span className={`status-badge ${alert.status}`}>{alert.status}</span>
+                      {isExpanded ? <MdExpandLess size={24} /> : <MdExpandMore size={24} />}
+                    </div>
+                  </div>
+
+                  <div className="alert-body">
+                    {alert.user_phone && <p><strong>Phone:</strong> {alert.user_phone}</p>}
+                    <p><strong>Location:</strong> {alert.latitude.toFixed(6)}, {alert.longitude.toFixed(6)}</p>
+                    <p><strong>Signal:</strong> {alert.signal_type}</p>
+                    <p><strong>Time:</strong> {new Date(alert.event_time).toLocaleString()}</p>
+                    {alert.notes && <p><strong>Notes:</strong> {alert.notes}</p>}
+                  </div>
+
+                  <div className="alert-actions">
+                    <textarea
+                      value={currentNotes}
+                      onChange={(e) => setNotes({ ...notes, [alert.id]: e.target.value })}
+                      placeholder="Add notes..."
+                      rows="2"
+                      onClick={(e) => e.stopPropagation()}
+                    />
+                    <div className="action-buttons">
+                      {alert.status === 'pending' && (
+                        <button
+                          onClick={(e) => {
+                            e.stopPropagation();
+                            handleStatusChange(alert.id, 'acknowledged', currentNotes);
+                          }}
+                          className="btn-acknowledge"
+                        >
+                          <FaCheck /> Acknowledge
+                        </button>
+                      )}
+                      {alert.status === 'acknowledged' && (
+                        <button
+                          onClick={(e) => {
+                            e.stopPropagation();
+                            handleStatusChange(alert.id, 'resolved', currentNotes);
+                          }}
+                          className="btn-resolve"
+                        >
+                          <FaCheckCircle /> Resolve
+                        </button>
+                      )}
+                    </div>
+                  </div>
+                </div>
+
+                {/* Expanded History */}
+                {isExpanded && (
+                  <div className="alert-history">
+                    <div className="history-header">
+                      <MdHistory /> Alert History ({history.length} total)
+                    </div>
+                    {isLoading ? (
+                      <p className="loading">Loading history...</p>
+                    ) : history.length > 0 ? (
+                      <div className="history-timeline">
+                        {history.map((histAlert, index) => (
+                          <div key={histAlert.id} className={`history-item ${histAlert.message_type}`}>
+                            <div className="history-marker">
+                              {index === 0 ? '📍' : '•'}
+                            </div>

+import React from 'react';
+import { FiUser, FiPhone, FiMapPin } from 'react-icons/fi';
+import './UserList.css';
+
+function UserList({ alerts }) {
+  const getAlertIcon = (messageType) => {
+    switch (messageType) {
+      case 'emergency': return '🔴';
+      case 'high': return '🟠';
+      case 'normal': return '🟢';
+      default: return '🔵';
+    }
+  };
+
+  const formatTimeAgo = (timestamp) => {
+    const now = new Date();
+    const alertTime = new Date(timestamp);
+    const diffMs = now - alertTime;
+    const diffMins = Math.floor(diffMs / 60000);
+    const diffHours = Math.floor(diffMins / 60);
+    const diffDays = Math.floor(diffHours / 24);
+
+    if (diffMins < 1) return 'Just now';
+    if (diffMins < 60) return `${diffMins} min ago`;
+    if (diffHours < 24) return `${diffHours} hour${diffHours > 1 ? 's' : ''} ago`;
+    return `${diffDays} day${diffDays > 1 ? 's' : ''} ago`;
+  };
+
+  return (
+    <div className="user-list">
+      <div className="user-list-header">
+        <h3>
+          <FiUser /> Personnel Status ({alerts.length})
+        </h3>
+      </div>
+      {alerts.length === 0 ? (
+        <p className="no-users">No active personnel</p>
+      ) : (
+        <div className="user-items">
+          {alerts.map((alert) => (
+            <div key={alert.device_id} className={`user-item ${alert.message_type}`}>
+              <div className="user-item-header">
+                <span className="alert-icon">{getAlertIcon(alert.message_type)}</span>
+                <div className="user-info">
+                  <div className="user-name">
+                    {alert.user_name || alert.device_id}
+                    {!alert.user_name && (
+                      <span className="unregistered-tag">⚠️ Unregistered</span>
+                    )}
+                  </div>
+                  <div className="user-device">{alert.device_id}</div>
+                  {alert.user_phone && (
+                    <div className="user-phone">
+                      <FiPhone size={12} /> {alert.user_phone}
+                    </div>
+                  )}
+                </div>
+              </div>
+              <div className="user-item-body">
+                <div className="alert-info">
+                  <span className={`alert-type-badge ${alert.message_type}`}>
+                    {alert.message_type.toUpperCase()}
+                  </span>
+                  <span className={`status-badge ${alert.status}`}>
+                    {alert.status}
+                  </span>
+                </div>
+                <div className="alert-time">
+                  {formatTimeAgo(alert.received_at)}
+                </div>
+                <div className="alert-location">
+                  <FiMapPin size={12} /> {alert.latitude.toFixed(4)}, {alert.longitude.toFixed(4)}
+                </div>
+              </div>
+            </div>
+          ))}
+        </div>
+      )}
+    </div>
+  );
+}
+
+export default UserList;
